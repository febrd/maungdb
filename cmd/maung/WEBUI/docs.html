MaungDB v2.2.4

MaungDB adalah Database Management System (DBMS) Hybrid yang menggabungkan kesederhanaan penyimpanan berbasis dokumen (flat-file) dengan ketatnya integritas data relasional (RDBMS).

Dibangun menggunakan bahasa Go (Golang), MaungDB menawarkan performa tinggi dengan bahasa kueri yang unik dan deklaratif berbasis Bahasa Sunda, menjadikannya sistem database yang "membumi" namun bertenaga.

Fitur Utama v2.2.4:
  • ACID Transactions (v2.2.3): Jaminan keamanan data dengan Write-Ahead Logging (WAL). Mendukung Commit & Rollback.
  • Indexing Engine (v2.2.4): Akses data secepat kilat (O(1)) menggunakan Hash Indexing dan Query Optimizer cerdas.
  • Strict Schema Enforcement: Validasi data ketat saat insert/update menggunakan definisi skema terpusat.
  • Constraint Engine: Dukungan penuh untuk Primary Key, Foreign Key (Relational Integrity), Unique, dan Not Null.
  • Relational Power: Mendukung INNER, LEFT, dan RIGHT JOIN untuk analisis data kompleks.

üèõÔ∏è Arsitektur & Konsep Internal

Memahami bagaimana MaungDB memproses data dari hulu ke hilir.

1. Storage Engine (Gudang Data)

MaungDB menggunakan pendekatan Hybrid Storage. Data disimpan dalam format teks yang dapat dibaca manusia (human-readable) namun dengan struktur biner yang terindeks secara logis.

  • File .mg (Data): Setiap tabel adalah satu file fisik. Baris data dipisahkan oleh newline (\n) dan kolom dipisahkan oleh pipe (|).
  • File .idx (Index): File binary map (JSON structure) untuk mempercepat pencarian data tanpa scanning seluruh file.
  • File wal.log (Journal): Log transaksi untuk mencatat perubahan sementara sebelum ditulis permanen (Commit).

2. The Execution Pipeline

Setiap query yang masuk (baik dari CLI maupun Web) melewati tahapan pipeline berikut:

üîê Keamanan & Autentikasi

MaungDB menerapkan sistem keamanan berbasis Role-Based Access Control (RBAC) yang ketat.

Hirarki Role (Tingkatan)

Role Hak Akses
supermaung God Mode. Bisa melakukan segalanya: membuat user, menghapus database, reset password, dan mengakses semua data tanpa batasan.
admin Schema Manager. Bisa membuat database (CREATEDB), membuat tabel (DAMEL), dan memodifikasi struktur schema. Tidak bisa memanajemen user lain.

üîê Transaksi Data (ACID) v2.2.3

Fitur ini menjamin integritas data dengan konsep All-or-Nothing. Menggunakan Write-Ahead Logging (WAL), perubahan data tidak akan ditulis ke disk sebelum dikonfirmasi (Commit).

MIMITIAN (Begin Transaction)

Memulai sesi transaksi. Semua perintah SIMPEN, OMEAN, atau MICEUN setelah perintah ini akan ditampung di memori sementara (WAL Buffer).

JADIKEUN (Commit)

Menyimpan permanen semua perubahan yang tertampung di memori ke dalam file fisik database.

BATALKEUN (Rollback)

Membatalkan semua perubahan yang belum di-commit. Data akan kembali ke kondisi semula sebelum MIMITIAN.

Contoh Skenario: Transfer Saldo

// 1. Mulai Transaksi
MIMITIAN

// 2. Potong saldo pengirim (Belum tersimpan permanen)
OMEAN nasabah JADI saldo=400000 DIMANA id=101

// 3. Tambah saldo penerima (Belum tersimpan permanen)
OMEAN nasabah JADI saldo=600000 DIMANA id=102

// 4. Jika ada error, user bisa ketik: BATALKEUN
// 5. Jika yakin, simpan permanen:
JADIKEUN
Catatan: Selama transaksi berlangsung, user lain mungkin tidak melihat perubahan data sampai Anda melakukan JADIKEUN (Isolation Level).

‚ö° Indexing & Optimasi v2.2.4

Fitur ini dirancang untuk mengatasi lambatnya pencarian data pada tabel besar (Full Table Scan). Dengan Indexing, MaungDB membuat peta pintas (Hash Map) untuk menemukan baris data secara instan (O(1)).

TANDAIN (Create Index)

Membuat file index (.idx) pada kolom tertentu.
Sinonim: TANDAAN, TAWISAN.

TANDAIN <nama_tabel> DINA <nama_kolom>

Cara Kerja:

  1. Saat index dibuat, sistem memetakan nilai kolom ke Primary Key.
  2. Ketika Anda menjalankan SIMPEN, index otomatis diperbarui di background (Async).
  3. Ketika Anda menjalankan TINGALI ... DIMANA kolom=nilai, Optimizer akan otomatis menggunakan index tersebut.

Contoh Penggunaan:

// 1. Buat Index pada kolom 'divisi' di tabel 'karyawan'
TANDAIN karyawan DINA divisi

// 2. Lakukan Query
TINGALI * TI karyawan DIMANA divisi=IT

Jika optimasi berhasil, pada log server/console akan muncul pesan:
‚ö° [OPTIMIZER] Index Scan on table 'karyawan', column 'divisi'

MaungDB v2.2.9 Enterprise

MaungDB v2.2.9 adalah rilis final untuk Core Engine (Local). Versi ini menutup celah fitur dengan kompetitor melalui penambahan Full Text Search (Inverted Index) dan Replication Logic Guard.

Dibangun dengan Go (Golang), MaungDB menggabungkan kecepatan NoSQL (Key-Value/Doc) dengan integritas SQL (ACID/Relational) dalam satu paket binary tunggal yang ringan.

Fitur Unggulan v2.2.9:
  • üîç Full Text Search (Korehan): Pencarian teks secepat kilat (O(1)) menggunakan Inverted Index.
  • üõ°Ô∏è Replication Guard (Indung/Anak): Logika Role-Based untuk persiapan sistem terdistribusi (Master-Slave).
  • ‚ö° ACID Transactions (WAL): Jaminan data aman (Commit/Rollback) dengan Write-Ahead Logging.
  • ü™û Virtual Views (Kaca): Simpan query kompleks sebagai tabel virtual.
  • ü§ñ Automation (Jarambah): Trigger otomatis saat Insert/Update/Delete.
  • üìä Advanced Analytics: Dukungan GROUP BY, HAVING, dan Agregasi lengkap.

üèõÔ∏è Arsitektur & Konsep Internal

MaungDB menggunakan pendekatan Hybrid Storage Engine:

  • Data (.mg): Penyimpanan baris data (Append-Only log structure).
  • Index (.idx): Hash Map binary untuk pencarian cepat (Exact Match).
  • Text Index (.fts): Inverted Index JSON untuk pencarian kata kunci (Full Text).
  • WAL (.log): Jurnal transaksi untuk menjamin durabilitas (ACID).
  • View (.view): Definisi query virtual.
